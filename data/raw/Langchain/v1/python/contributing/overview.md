## 贡献方式

### 报告 Bug

发现 Bug 了吗？请按照以下步骤帮助我们修复它：

1.  **搜索**
    检查相应仓库的 GitHub Issues 中是否已存在该问题：
    *   [LangChain Issues](https://github.com/langchain-ai/langchain/issues)
    *   [LangGraph Issues](https://github.com/langchain-ai/langgraph/issues)

2.  **创建 Issue**
    如果不存在 Issue，请创建一个新的。编写时，请务必遵循提供的模板，并包含一个[最小化、可复现的示例](https://stackoverflow.com/help/minimal-reproducible-example)。创建完成后，附加任何相关的标签到最终的 Issue 上。如果项目维护者无法重现该问题，它可能不会被及时处理。

3.  **等待**
    项目维护者将对 Issue 进行分类并可能要求提供额外信息。由于我们需要处理大量 Issue，请耐心等待。除非您有新的信息可提供，否则请勿催促 Issue。

如果您要添加 Issue，请尽量使其专注于一个主题。如果两个 Issue 相关或互为阻碍，请[链接它们](https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/linking-a-pull-request-to-an-issue#linking-a-pull-request-to-an-issue-using-a-keyword)，而不是将它们合并。例如：

```
This issue is blocked by #123 and related to #456.
```

### 建议功能

对新功能或增强功能有想法吗？

1.  **搜索**
    在相应的仓库 Issue 中搜索现有功能请求：
    *   [LangChain Issues](https://github.com/langchain-ai/langchain/issues?q=state%3Aopen%20label%3A%22feature%20request%22)
    *   [LangGraph Issues](https://github.com/langchain-ai/langgraph/issues?q=state%3Aopen%20label%3Aenhancement)

2.  **讨论**
    如果不存在请求，请在[相关类别](https://forum.langchain.com/c/help/langchain/14)下发起新的讨论，以便项目维护者和社区可以提供反馈。

3.  **描述**
    务必描述用例以及它对其他人有价值的原因。如果可能，请提供适用的示例或模型。概述应该通过的测试用例。

*   [如何创建您的第一个拉取请求 (Pull Request) 的指南](https://langchain-doc.cn/v1/python/contributing/code)

如果您开始处理一个 Issue，请将其分配给自己或请求维护者为您分配。这有助于避免重复工作。

如果您正在寻找可处理的任务，请查看我们仓库中标记为 "**good first issue**" 或 "**help wanted**" 的 Issue：

*   [LangChain 标签](https://github.com/langchain-ai/langchain/labels)
*   [LangGraph 标签](https://github.com/langchain-ai/langgraph/labels)
